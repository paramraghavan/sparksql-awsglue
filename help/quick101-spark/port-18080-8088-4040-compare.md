# Spark Ports: UI-4040, History-18080, and YARN-8088


Navigating the different Spark web interfaces can feel like a game of musical chairs with port numbers. Each port serves
a specific stage of an application's lifecycle or a specific part of the cluster architecture.

Here is the breakdown of the three most common Spark ports:

---

### **1. Port 4040: The Spark UI (The "Live" View)**

This is the port you’ll visit most often while your code is actually running. It provides a real-time look into the
specific SparkContext of a running application.

* **Scope:** Single Application.
* **Availability:** Only while the application is **running**. Once the `SparkContext` stops (or your script finishes),
  this UI disappears.
* **Key Info:** Stages, storage (cached data), environment variables, and active executors.
* **Note:** If you start a second Spark application on the same machine, it will try 4041, then 4042, and so on.

---

### **2. Port 18080: The Spark History Server**

Since port 4040 dies when your job finishes, you need a way to see what happened after the fact. This is where the
History Server comes in.

* **Scope:** All completed (and sometimes incomplete) applications.
* **Availability:** Always (if the History Server daemon is running).
* **How it works:** It reads event logs that Spark writes to a file system (like HDFS or S3) during execution.
* **Use Case:** Debugging a job that failed overnight or reviewing performance trends of past runs.

---

### **3. Port 8088: The YARN ResourceManager UI**

This port isn't actually "Spark"—it belongs to **Hadoop YARN**. However, since most enterprise Spark jobs run on top of
YARN, this is your "bird's-eye view" of the entire cluster.

* **Scope:** The entire Cluster.
* **Availability:** Always (as long as the Hadoop cluster is up).
* **Key Info:** Shows all applications (Spark, MapReduce, Flink) competing for resources. It tells you how much memory
  and how many CPU cores are being used across the whole fleet.
* **The Link:** Usually, if you are in the YARN UI (8088), you can click a link labeled **"Tracking UI"** for a specific
  job, and it will redirect you to either the 4040 UI (if running) or the 18080 UI (if finished).

---

### **Quick Comparison Table**

| Port      | Service            | What you see                         | When to use it                                       |
|:----------|:-------------------|:-------------------------------------|:-----------------------------------------------------|
| **4040**  | **Spark UI**       | Direct details of a **running** job. | "Is my current job stuck on a specific stage?"       |
| **18080** | **History Server** | Details of **past** jobs.            | "Why did my job fail two hours ago?"                 |
| **8088**  | **YARN UI**        | Status of the **entire cluster**.    | "Is the cluster full? How many resources did I get?" |

### **A Pro-Tip on Access**

If you are running Spark on a cloud provider (like EMR, Databricks, or Google Cloud Dataproc), these ports might be
blocked by firewalls or mapped to different URLs. You often have to use a "Quick Link" in the cloud console or set up an
SSH tunnel to see them.
