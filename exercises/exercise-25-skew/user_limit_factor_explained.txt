Q: If I set user-limit-factor = 100, will I block other users?
A: NO! Here's how it works:

SCENARIO 1: You're the only user
â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”
You request: 240 executors
You get:     240 executors (100% of cluster)
Other users: None
Result:      You use everything, nothing is wasted âœ“


SCENARIO 2: Another user submits a job while yours is running
â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”
You:    Have 240 executors (100% of cluster)
User2:  Submits job requesting 120 executors

What happens:
1. Your job continues running with 240 executors
2. As your tasks complete and executors are released, User2 gets them
3. Eventually settles at: You=120, User2=120 (fair sharing)
4. When User2 finishes, you can scale back to 240

Result: Dynamic fair sharing, everyone gets resources âœ“


SCENARIO 3: Both users submit at same time
â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”
You:    Request 240 executors
User2:  Requests 120 executors

What happens:
YARN allocates proportionally:
You get:    160 executors (240 / 360 = 67%)
User2 gets: 80 executors (120 / 360 = 33%)

Result: Fair sharing based on demand âœ“


KEY POINT:
â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”â”

"If the cluster is available, one user can use it all"
"If multiple users need resources, share fairly"
